## Readiness Summary

This file is the release-readiness artifact for the stable `0.4.0` release. It
records the command-backed proof for the release basis and the post-publish
verification for the public crates.io artifact.

Release execution completed with the following posture:

- tag `v0.4.0` points at `b3ebe6623b91ca372040964e1f94af25f081e3b7`,
- all 13 public crates were published and verified by the Release workflow,
- public versioned and unversioned `cargo install shipper` smoke tests passed,
- the GitHub Release exists with final `.shipper` evidence attached, and
- the `x86_64-apple-darwin` binary asset remains a workflow carry-over because
  that build leg failed to install the target on the tag run.

Current product claim state lives in `docs/status/SUPPORT_TIERS.md`. This
readiness record does not promote Trusted Publishing default, live crates.io
yank/fix-forward execution, signing, SBOM, SLSA provenance, or alternative
registry profiles.

### Local Full-Suite Caveat

The local Windows proof worktree did not produce a clean full-suite
`cargo test --workspace --all-features --locked` pass. Two full runs exposed
localhost mock-registry connection-refused races, and one run also observed the
git-cleanliness dirty-state unit test returning clean under parallel execution.
The exact failing tests passed when rerun individually:

- `cargo test -p shipper-cli --test bdd_error_handling ambiguous_resolves_via_registry::given_ambiguous_failure_when_registry_shows_published_then_marked_published -- --exact --nocapture`
- `cargo test -p shipper-cli --test bdd_resume resume_completed_state::given_all_published_state_when_resume_then_succeeds_without_publish -- --exact --nocapture`
- `cargo test -p shipper-core ops::git::cleanliness::tests::ensure_git_clean_new_phrasing -- --exact --nocapture`
- `cargo test -p shipper-core engine::tests::test_resume_from_skips_initial_packages -- --exact --nocapture`

A serial full-suite retry exceeded a 50-minute local timeout and was stopped.
Local `cargo nextest run --workspace --all-features --locked` reproduced two
mock-registry connection races; both exact failures passed individually. The
release relied on the final GitHub PR CI matrix for the full nextest gate before
tagging.

## Known Carry-Over

- Trusted Publishing default remains planned/advisory until crates.io-side
  registration and short-lived-token release proof exist for the full published
  crate set.
- Live crates.io yank/fix-forward execution is not promoted beyond the bounded
  dry-run/fake-Cargo proof surfaces in this release.
- Receipt signing, SBOM generation, and SLSA-style provenance remain future
  work.
- crates.io is the proof-backed registry profile in this release; additional
  registries remain follow-up work.
- Broader mutation coverage and further CI cost routing remain future work.
- Live-runner interruption proof was not part of this readiness packet's
  evidence base; current support tiers now record GitHub Actions run
  `26051581056` and the `shipper-live-interruption-seed-26051581056` /
  `shipper-live-interruption-resume-26051581056` artifacts as the later
  stable/internal proof for artifact handoff and safe resume against fake
  Cargo/mock registry surfaces.
- Local Windows full-suite execution exposed mock-registry harness instability;
  final tag approval should require green GitHub PR CI on the readiness PR.
- The release workflow did not produce the `x86_64-apple-darwin` binary asset
  for `v0.4.0`; crates.io installation remains the supported path for Intel
  macOS users until the release binary workflow is corrected and the asset is
  recovered or a later patch release is cut.
