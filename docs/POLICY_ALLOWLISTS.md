# Policy Allowlists

This document is the index for all policy allowlist files in `shipper`. Each allowlist is a TOML file in `policy/` with a defined schema and purpose.

## Allowlist Files

| File | Purpose | Commands |
|---|---|---|
| `policy/clippy-lints.toml` | Active and planned Clippy lint ledger | `cargo xtask check-lint-policy` |
| `policy/clippy-debt.toml` | Receipted Clippy suppression debt with owner + expiry | `cargo xtask check-clippy-exceptions` |
| `policy/clippy-exceptions.toml` | Permanent Clippy exceptions with business justification | `cargo xtask check-clippy-exceptions` |
| `policy/no-panic-allowlist.toml` | Permanently receipted panic-family calls (true invariants) | `cargo xtask check-no-panic-family` |
| `policy/no-panic-baseline.toml` | Debt snapshot of existing panic-family calls (generated) | `cargo xtask check-no-panic-family` |
| `policy/non-rust-allowlist.toml` | General non-Rust files (docs, configs, templates) | `cargo xtask check-file-policy` |
| `policy/generated-allowlist.toml` | Machine-generated files (snapshots, baselines) | `cargo xtask check-generated` |
| `policy/executable-allowlist.toml` | Files with execute permission | `cargo xtask check-executable-files` |
| `policy/dependency-surface-allowlist.toml` | Dependency metadata and lock files | `cargo xtask check-dependency-surfaces` |
| `policy/workflow-allowlist.toml` | GitHub Actions workflow files | `cargo xtask check-workflow-surfaces` |
| `policy/process-allowlist.toml` | Shell commands and subprocesses in workflows | `cargo xtask check-process-policy` |
| `policy/network-allowlist.toml` | External network endpoints in workflows | `cargo xtask check-network-policy` |
| `policy/ripr-suppressions.toml` | Suppressed ripr exposure findings | `cargo xtask ripr-pr` |
| `policy/ci-budget.toml` | CI time/resource budgets per lane | `cargo xtask ci plan` |
| `policy/ci-lanes.toml` | Lane definitions and trigger rules | `cargo xtask ci plan` |
| `policy/ci-risk-packs.toml` | Risk-pack definitions for scoped PR routing | `cargo xtask ci plan` |

## Allowlist Schema Conventions

All allowlist files use TOML with `schema_version = "1.0"` at the top level.

### Clippy Lint Entry

```toml
[[active]]
name = "clippy::dbg_macro"
level = "deny"
class = "hygiene"
reason = "Debug macros are not a reviewable diagnostics path."

[[planned]]
name = "clippy::manual_checked_ops"
level = "warn"
activate_when_msrv = "1.95"
reason = "Prefer checked arithmetic over manual guards."
```

### Clippy Debt Entry

```toml
[[debt]]
name = "clippy::needless_pass_by_value"
crate = "shipper-core"
path = "crates/shipper-core/src/plan/mod.rs"
owner = "team"
expiry = "2026-09-01"
reason = "Refactor planned in API cleanup PR."
```

### No-Panic Baseline Entry

```toml
[[entry]]
path = "crates/shipper-core/src/engine/publish.rs"
family = "unwrap"
selector_kind = "method_call"
selector_callee = "unwrap"
snippet = "lock.read().unwrap()"
count = 1
```

### Non-Rust Allowlist Entry

```toml
[[file]]
path = "codecov.yml"
kind = "config"
reason = "Codecov integration configuration."

[[glob]]
pattern = "docs/**/*.md"
kind = "documentation"
reason = "Project documentation."
```

### Workflow Allowlist Entry

```toml
[[workflow]]
path = ".github/workflows/release.yml"
reason = "Primary release workflow using Trusted Publishing."
process_policy = "release"
network_policy = "release"
```

### Process Policy Entry

```toml
[[policy]]
name = "release"
allowed_processes = ["cargo", "rustup", "shipper", "gh", "tar", "sha256sum"]
```

### Network Policy Entry

```toml
[[policy]]
name = "release"
allowed_endpoints = [
  "crates.io",
  "static.crates.io",
  "token.actions.githubusercontent.com",
  "api.github.com",
]
```

## Full Policy Report

The consolidated policy report is generated by:

```bash
cargo xtask policy-report
```

It includes:
- MSRV alignment status (Cargo.toml / clippy.toml / policy files)
- Active and planned Clippy lints with activation status
- Clippy debt count and oldest expiry
- No-panic baseline size and recent delta
- File policy coverage percentage
- ripr status (if available)
- CI budget utilization (if available)

## Suppressions and Exceptions

Suppressions in code use `#[expect(lint_name, reason = "...")]`. Bare `#[allow(lint_name)]` without a reason is a policy violation caught by `cargo xtask check-clippy-exceptions`.

Every suppression in code must have a corresponding entry in `policy/clippy-debt.toml` or `policy/clippy-exceptions.toml`. The distinction:

- **Debt**: Temporary. Has an owner and expiry. Will be fixed.
- **Exception**: Permanent. Has a business justification. Reviewed annually.

## Rollout Status

The 0.4.0 quality rollout is split into a ladder of small, separately reviewable PRs rather than one large policy-infrastructure PR. The full tracker for the non-Rust file-policy subset (PRs 1–12 below) lives in [docs/policy/NON_RUST_ROLLOUT.md](policy/NON_RUST_ROLLOUT.md). The broader 0.4.0 rollout (Clippy, no-panic, ripr, CI lanes, release dry-run) is tracked under [#109](https://github.com/EffortlessMetrics/shipper/issues/109).

A receipt in any of these allowlists is **not** "approved forever." It is "known surface, owner, reason, and current disposition." A valid `reason` may include `"scheduled to be converted to Rust/xtask"` when the file exists for legacy compatibility or migration staging, paired with an `expires` date.
